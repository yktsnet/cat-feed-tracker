# LINE リッチメニューと Webhook 連携仕様

本ドキュメントでは、LINE Messaging API との連携設定、リッチメニューのボタン配置、および関連する操作仕様について説明します。

---

## 1. LINE Messaging API 設定

1. [LINE Developers Console](https://developers.line.biz/) にログインし、プロバイダおよび Messaging API チャネルを作成します。
2. チャネル設定で Webhook を有効にし、URL に `https://your-domain.example.com/api/webhook/line` を指定します。
3. チャネルシークレット (`LINE_CHANNEL_SECRET`) とチャネルアクセストークン（長期 - `LINE_CHANNEL_ACCESS_TOKEN`）を取得し、サーバー側の `.env` に設定します。

---

## 2. リッチメニューのボタン動作フロー

本プロジェクトのリッチメニューは 2×2 のグリッド構成で、各領域をタップするとユーザーから特定のテキストメッセージが自動送信され、Webhook 経由でサーバーが応答します。

| ボタン | 送信テキスト | 処理・レスポンス内容 |
|---|---|---|
| **今日の記録** | `今日の記録` | 現時点までの当日の給餌ログを一覧で返します。 |
| **平均** | `平均` | 直近4週間の週平均と、直近3ヶ月の月平均（給餌イベントがあった日のみを有効日として計算）を返します。 |
| **体重** | `体重` | 猫の選択（`config/cats.yaml` に定義した猫名が番号付きで表示）および体重数値入力の2ステップ対話を開始し、記録完了後に履歴5件と給餌平均を表示します。 |
| **設定** | `設定` | 現在の通知のオン/オフ状態および上限警告回数を表示し、変更コマンドを案内します。 |

### 追加テキストコマンド
- **`通知オン`** / **`通知オフ`**：通知の有効/無効を直接切り替えます。
- **`上限変更`**：1日の上限警告回数の変更対話（回数入力待ち）を開始します。
- **`キャンセル`**：任意の対話入力待ちステートから抜け、処理をキャンセルします。

---

## 3. リッチメニューの登録手順

提供されている `setup_rich_menu.py` スクリプトを実行することで、LINE 公式アカウントに 2×2 のリッチメニュー画像をアップロードし、アクションを設定できます。

### 実行方法

```bash
# NixOS 環境の場合
nix-shell -p python3Packages.pillow --run "python server/scripts/setup_rich_menu.py"

# その他の環境の場合
pip install Pillow
LINE_CHANNEL_ACCESS_TOKEN=<あなたのチャネルアクセストークン> python server/scripts/setup_rich_menu.py
```
